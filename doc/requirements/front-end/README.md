# Frontend Requirements

User stories with acceptance criteria and embedded test plans for all user-facing task management clients (web and mobile). All stories document **implemented** functionality unless noted.

Acceptance criteria use platform-neutral wording. Backend and frontend epic numbers differ by design (e.g. BE Epic 1 = CRUD API, FE Epic 1 = task list).

- **Test plan convention:** [ADR 003](../../adr/003-test-plan-integration-in-user-stories.md)
